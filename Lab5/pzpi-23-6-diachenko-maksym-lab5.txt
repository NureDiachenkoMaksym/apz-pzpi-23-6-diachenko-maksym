2 КОРОТКИЙ ОПИС ПРОЄКТУ

Програмна система аналізу природних ресурсів призначена для централізованого обліку, перевірки, аналізу та візуалізації даних про стан природних ресурсів регіону. Система підтримує роботу з лісовими, водними та ґрунтовими показниками, дозволяє створювати спостереження, переглядати аналітичну панель, експортувати дані та масштабувати backend-частину.

3 ЗАПЛАНОВАНІ ТА ВИКОНАНІ ФУНКЦІЇ

Заплановано:
- серверна частина з REST API;
- база даних для спостережень;
- web-застосунок із користувацьким та адміністративним режимами;
- мобільний застосунок для польових спостережень;
- підтримка локалізації;
- масштабування backend;
- демонстрація навантажувального тестування.

Виконано:
- реалізовано backend API на FastAPI;
- створено структуру бази даних спостережень;
- реалізовано web-клієнт з українською та англійською локалізацією;
- підготовлено Android Kotlin-код мобільного клієнта;
- створено Docker Compose конфігурацію;
- підготовлено Nginx load balancer;
- підготовлено Kubernetes manifests;
- підготовлено Locust-сценарій навантажувального тестування;
- оформлено матеріали для GitHub-репозиторію.

Не виконано в межах поточного навчального прототипу:
- повноцінна авторизація через реальний identity provider;
- повне підключення до реальних державних відкритих наборів даних;
- промислове розгортання в хмарному середовищі;
- навчання власної моделі штучного інтелекту.

4 МАТЕРІАЛИ ПРЕЗЕНТАЦІЇ

Презентація готового проєкту повинна містити короткий опис проблеми, мету системи, основні компоненти, демонстрацію web-клієнта, мобільного клієнта, backend API, масштабування та висновки.
