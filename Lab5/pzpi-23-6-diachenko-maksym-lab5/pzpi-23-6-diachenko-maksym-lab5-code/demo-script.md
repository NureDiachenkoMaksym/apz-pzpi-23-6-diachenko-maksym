# Сценарій відео-демо 4–6 хвилин

00:00 — Назва проєкту та проблема. Дані про природні ресурси часто зберігаються в різних таблицях і звітах, тому потрібна єдина система обліку та аналізу.

00:40 — Загальна архітектура. Система складається з backend API, бази даних, web-клієнта, мобільного клієнта та конфігурацій масштабування.

01:20 — Web-клієнт. Демонструється аналітична панель, таблиця спостережень, фільтр за типом ресурсу, українська та англійська локалізація.

02:10 — Адміністративний режим. Показується управління даними, експорт, імпорт і створення резервної копії.

03:00 — Мобільний клієнт. Пояснюється сценарій польового оператора: перегляд даних, створення спостереження, синхронізація з сервером.

03:50 — Backend API. Демонструються endpoint-и /health, /api/observations і /api/dashboard/summary.

04:40 — Масштабування. Пояснюється Docker Compose, Nginx load balancer, Kubernetes Deployment і Locust-тестування.

05:30 — Висновки. Пояснюється, що виконано, а що може бути розширено в майбутньому: реальні датасети, повна авторизація, ШІ-модулі прогнозування.
