2 ЗАВДАННЯ

Метою лабораторної роботи є демонстрація масштабування backend-частини програмної системи аналізу природних ресурсів для роботи із великим навантаженням. Для цього обрано горизонтальне масштабування серверної частини з використанням кількох копій API-сервера, балансувальника навантаження та спільної бази даних.

3 СТРАТЕГІЯ МАСШТАБУВАННЯ

Backend-сервіс реалізовано як stateless REST API. Це означає, що сервер не зберігає користувацький стан у пам’яті конкретного процесу, а всі довготривалі дані зберігаються в базі даних. Завдяки цьому можна запускати кілька однакових екземплярів API-сервера та розподіляти запити між ними.

У демонстраційній конфігурації використано:
- FastAPI як backend API;
- PostgreSQL як централізовану базу даних;
- Nginx як load balancer;
- Docker Compose для локального запуску;
- Kubernetes manifests для демонстрації масштабування у кластері;
- Locust для навантажувального тестування.

4 ТЕХНІЧНІ РІШЕННЯ

Масштабування можливе завдяки таким рішенням:
- серверна частина не зберігає стан сесій локально;
- база даних винесена в окремий сервіс;
- API має health-check endpoint /health;
- Nginx розподіляє запити між копіями API;
- Docker-образ backend може запускатися у кількох екземплярах;
- Kubernetes Deployment дозволяє змінювати кількість реплік.

5 АНАЛІЗ ВУЗЬКИХ МІСЦЬ

Першим вузьким місцем при збільшенні навантаження може стати база даних, оскільки всі API-сервери звертаються до неї. Другим обмеженням може бути пропускна здатність мережі між балансувальником і API-серверами. Також важливими є обмеження CPU та пам’яті контейнерів.

Для зменшення ризиків передбачено кешування довідкових даних, індекси в базі даних, пагінацію списків, обмеження розміру імпорту та відокремлення важких аналітичних задач від звичайних API-запитів.

6 НАВАНТАЖУВАЛЬНЕ ТЕСТУВАННЯ

Для тестування підготовлено Locust-сценарій. Він виконує запити до основних endpoint-ів: /health, /api/dashboard/summary, /api/observations та POST /api/observations.

Очікувана залежність: зі збільшенням кількості реплік API зростає кількість запитів за секунду, яку витримує система, доки вузьким місцем не стане база даних або балансувальник.

7 ВИСНОВКИ

У лабораторній роботі було розроблено демонстраційну backend-архітектуру, що підтримує горизонтальне масштабування. Було підготовлено Docker Compose, Nginx load balancer, Kubernetes manifests та Locust-сценарій для навантажувального тестування.
