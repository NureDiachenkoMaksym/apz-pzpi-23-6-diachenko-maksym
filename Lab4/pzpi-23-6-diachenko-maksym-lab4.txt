2 ЗАВДАННЯ

Метою лабораторної роботи є демонстрація способу масштабування backend частини програмної системи для роботи з великим навантаженням.

3 ОПИС ВИКОНАНОЇ РОБОТИ

Для системи аналізу природних ресурсів обрано горизонтальне масштабування backend API. Кілька копій серверного застосунку виконують однакові функції та обслуговують запити користувачів паралельно. Перед ними розташований load balancer, який розподіляє трафік між активними екземплярами API.

Технічні рішення:
- контейнеризація backend API за допомогою Docker;
- деплой у Kubernetes через Deployment;
- доступ до сервісу через Kubernetes Service типу LoadBalancer;
- автоматичне масштабування через Horizontal Pod Autoscaler;
- використання PostgreSQL як централізованого сховища даних;
- навантажувальне тестування за допомогою Locust.

Вузькі місця системи:
- CPU backend API при великій кількості одночасних запитів;
- кількість доступних з’єднань до бази даних;
- швидкість виконання складних аналітичних запитів;
- пропускна здатність мережі;
- обсяг оперативної пам’яті при кешуванні великих наборів даних.

Результати умовного навантажувального тестування показують, що збільшення кількості реплік backend API підвищує кількість запитів за секунду, яку може обробити система.

4 ВИСНОВКИ

У результаті виконання лабораторної роботи було запропоновано стратегію горизонтального масштабування backend частини програмної системи аналізу природних ресурсів. Було підготовлено Dockerfile, Kubernetes-конфігурації, налаштування сервісу, HPA та файл навантажувального тестування Locust.
