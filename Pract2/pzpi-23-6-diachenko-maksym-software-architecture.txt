2 ЗАВДАННЯ

Метою практичного заняття є дослідження архітектури відомої складної програмної системи. Для аналізу обрано Google Earth Engine, оскільки ця система є хмарною платформою для роботи з великими обсягами геопросторових і супутникових даних та безпосередньо пов’язана з тематикою аналізу природних ресурсів.

3 ОПИС ВИКОНАНОЇ РОБОТИ

3.1 Загальні відомості про систему

Google Earth Engine — це хмарна платформа для оброблення супутникових знімків, геопросторових даних та побудови аналітичних сценаріїв. Система використовується в екологічному моніторингу, лісовому господарстві, аналізі водних ресурсів, аграрному секторі та дослідженнях змін клімату.

3.2 Архітектурний стиль

Система має багаторівневу хмарну архітектуру з клієнт-серверною взаємодією, API-рівнем, розподіленим сховищем даних та серверним виконанням обчислень. Користувач описує обчислення через JavaScript API або Python API, а реальна обробка виконується на серверній інфраструктурі.

3.3 Основні компоненти

Основними компонентами є вебредактор коду, API, каталог геопросторових даних, обчислювальні сервіси, система візуалізації та хмарна інфраструктура. Важливою властивістю є відкладене виконання: обчислення описуються на клієнті, але виконуються сервером лише після запиту результату.

3.4 Масштабованість

Масштабованість забезпечується за рахунок розподіленої обробки даних і централізованого каталогу з попередньо підготовленими наборами даних. Це дозволяє аналізувати великі території та довгі часові ряди без локального завантаження супутникових знімків.

3.5 Практичний приклад

У піддиректорії Pract2/pzpi-23-6-diachenko-maksym-software-architecture/pzpi-23-6-diachenko-maksym-software-architecture-code розміщено приклад коду для розрахунку NDVI. Такий приклад демонструє архітектурний принцип, коли клієнтський код формує обчислювальний граф, а серверна платформа виконує оброблення даних.

4 ВИСНОВКИ

У результаті виконання практичного заняття було проаналізовано архітектуру Google Earth Engine. Система є прикладом масштабованої хмарної платформи для геопросторової аналітики. Її архітектурні принципи можуть бути використані як орієнтир для проєктування програмної системи аналізу природних ресурсів.
