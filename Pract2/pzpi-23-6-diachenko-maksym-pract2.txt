3 ОПИС ВИКОНАНОЇ РОБОТИ

3.1 Загальні відомості про систему

Назва системи: Google Earth Engine.

Розробник: Google.

Google Earth Engine — це хмарна програмна платформа для зберігання, оброблення та аналізу великих обсягів геопросторових даних. Система використовується для аналізу супутникових знімків, оцінювання стану природних ресурсів, моніторингу лісів, водойм, ґрунтів, сільськогосподарських територій та змін навколишнього середовища.

Користувачами системи є дослідники, екологи, географи, аналітики, державні організації, освітні установи та розробники програмних рішень у сфері геоінформаційних технологій.

Основні функції Google Earth Engine:
- доступ до великих наборів супутникових і геопросторових даних;
- хмарне оброблення даних;
- створення алгоритмів аналізу;
- візуалізація результатів на карті;
- обчислення індексів, наприклад NDVI;
- експорт результатів для подальшого використання.


3.2 Архітектурний стиль системи

Google Earth Engine має багаторівневу хмарну архітектуру. У системі поєднано такі архітектурні підходи:
- клієнт-серверна архітектура;
- сервісно-орієнтована архітектура;
- розподілена хмарна обробка даних;
- модульна архітектура;
- архітектура з віддаленим виконанням обчислень.

Клієнтська частина використовується для написання коду, створення запитів та відображення результатів. Основні обчислення виконуються не на комп’ютері користувача, а на серверах Google. Це дозволяє обробляти великі обсяги супутникових даних без необхідності завантажувати їх локально.


3.3 Основні компоненти системи

До основних компонентів Google Earth Engine належать:

1. Клієнтський інтерфейс.
Користувач взаємодіє із системою через вебредактор коду, Python API або JavaScript API. Через ці засоби формуються запити до хмарної інфраструктури.

2. API-рівень.
API забезпечує взаємодію між клієнтською частиною та серверною частиною. Він приймає запити користувача, передає їх на виконання та повертає результати.

3. Каталог геопросторових даних.
Система містить великі набори супутникових знімків, кліматичних даних, даних про земний покрив, рельєф, водні ресурси та інші природні об’єкти.

4. Серверна частина оброблення.
Серверна частина відповідає за виконання обчислень, фільтрацію даних, агрегацію, побудову індексів та аналіз зображень.

5. Система візуалізації.
Результати аналізу можуть бути представлені у вигляді картографічних шарів, графіків, статистичних значень або експортованих наборів даних.

6. Хмарна інфраструктура.
Обчислення виконуються у розподіленому середовищі Google, що забезпечує масштабованість, швидкість оброблення та можливість роботи з великими даними.


3.4 Взаємодія компонентів

Взаємодія компонентів Google Earth Engine відбувається за таким принципом:

1. Користувач створює скрипт у вебредакторі або через API.
2. Клієнтська частина формує запит до серверної частини.
3. Серверна частина визначає необхідні набори даних з каталогу.
4. Обчислювальні сервіси виконують фільтрацію, оброблення та аналіз.
5. Результати повертаються клієнту у вигляді карти, статистики або файлів для експорту.

Важливою особливістю є те, що Google Earth Engine використовує відкладене виконання обчислень. Користувач описує послідовність операцій, але фактичне виконання відбувається на сервері після запуску обчислення або візуалізації результату.


3.5 Масштабованість та надійність

Google Earth Engine є масштабованою системою, оскільки використовує хмарну інфраструктуру та розподілену обробку даних.

Основні механізми масштабованості:
- виконання обчислень на серверній стороні;
- розподіл задач між обчислювальними вузлами;
- використання централізованого каталогу даних;
- оптимізація запитів;
- кешування результатів;
- можливість оброблення великих територій та тривалих часових періодів.

Надійність системи забезпечується використанням хмарної інфраструктури Google, зберіганням даних у централізованих сховищах та можливістю повторного виконання обчислень.


3.6 Приклади програмної реалізації

Для демонстрації роботи системи було підготовлено приклад програмного коду для Google Earth Engine JavaScript API. Код демонструє обчислення індексу NDVI для аналізу стану рослинності.

NDVI використовується для оцінювання рослинного покриву. У програмній системі аналізу природних ресурсів такий підхід може застосовуватися для:
- моніторингу стану лісів;
- оцінювання сільськогосподарських територій;
- виявлення змін рослинності;
- аналізу наслідків посухи;
- спостереження за деградацією природних ресурсів.

Google Earth Engine є прикладом складної програмної системи, яка дозволяє виконувати аналіз великих обсягів даних без локального зберігання та оброблення. Такий підхід є особливо корисним для задач моніторингу природних ресурсів, аналізу супутникових знімків та дослідження змін довкілля.
